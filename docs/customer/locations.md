# Locations

Locations (also called **Sites**) let you organize your devices by physical place — a stable, a greenhouse, a building. This makes it easier to find devices and assign different access policies per location.

## Locations page

The Locations page is the default landing page after login. It shows all your sites with a device count for each.

> **Screenshot:** *Locations page with search bar and two site cards: "Stable A" with icon, address "Farmroad 12, Dronten", 3 devices; "Greenhouse 1" with 1 device. A large "+" button for adding a new location.*

Click on a site card to see only the devices in that location.

## Add a location

1. Click the **+** button on the Locations page
2. Enter the site name (required)
3. Enter the address (optional — helps identify the physical location)
4. Save

> **Screenshot:** *Modal dialog "New Location" with name field "Stable A" and address field "Farmroad 12, Dronten, Netherlands". Cancel and Save buttons.*

## Assign a device to a location

You can assign a device to a site in two ways:

**During device activation** (claim wizard step 3):
- Select the site from the list when claiming a new device

**From the device dashboard**:
- Open the device → settings → change site assignment

## Unassigned devices

Devices without a site assignment appear under **"Unassigned"** in the device list. Use the sidebar filter `?site=unassigned` to find them.

## Access control per location

Site-level access policies let admins control exactly what each user can do in each location. This is useful when different staff are responsible for different buildings.
