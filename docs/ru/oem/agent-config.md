# Конфигурация агента

Агент MYXON читает конфигурацию из `/opt/myxon-agent/agent.env`. Это простой файл вида ключ=значение, загружаемый при старте.

## Полный справочник конфигурации

```ini
# ─── Обязательное ────────────────────────────────────────────────────────────

# URL облачного сервера MYXON (должен быть достижим с устройства)
MYXON_CLOUD_URL=https://myxon.yourcompany.com

# ─── Поток регистрации (выберите один) ───────────────────────────────────────

# Вариант A — Код активации (Сценарий 2 / OEM SDK)
# Задаётся один раз перед первым запуском. Агент израсходует этот код при первом
# подключении и сохранит полученную identity в MYXON_DEVICE_STATE.
# После первого запуска это значение игнорируется (приоритет у device.json).
MYXON_ACTIVATION_CODE=A3F1-B2E4-C9D7-0F56

# Вариант B — Пред-зарегистрированный серийник (Сценарий 1 / legacy)
# Используется, когда серийник устройства вручную зарегистрирован в Портале дилера.
# MYXON_SERIAL=MX-2026-00001

# ─── Сохранение состояния ─────────────────────────────────────────────────────

# Где хранится identity активированного устройства (device_id, серийник, frpc-токен).
# Создаётся агентом автоматически при первой успешной активации.
# НЕ удаляйте этот файл, если не хотите сбросить регистрацию устройства.
MYXON_DEVICE_STATE=/etc/myxon/device.json

# ─── Обнаружение в LAN ─────────────────────────────────────────────────────────

# Режим сканирования — как агент находит контроллеры в LAN:
#
#   auto (по умолчанию)
#     Сначала сканирует не-WAN интерфейсы (выделенный порт LAN).
#     Откатывается к сканированию подсети WAN-интерфейса, если выделенного LAN нет.
#     Лучше для: Orange Pi в существующей ферм-LAN (один кабель до коммутатора).
#
#   lan-only
#     Сканирует только не-WAN интерфейсы. Никогда не трогает подсеть WAN.
#     Лучше для: Orange Pi как роутер/шлюз с раздельными eth0 (WAN) и eth1 (LAN).
#
#   all
#     Сканирует все интерфейсы, включая WAN.
#     Использовать, когда: одноинтерфейсное устройство, где WAN И ЕСТЬ ферм-LAN.
#
MYXON_SCAN_MODE=auto

# ─── Тайминги ──────────────────────────────────────────────────────────────────

# Как часто слать heartbeat серверу (секунды)
MYXON_HEARTBEAT_INTERVAL=15

# Как часто пере-сканировать LAN на новые контроллеры (секунды)
MYXON_DISCOVERY_INTERVAL=60

# ─── Пути ────────────────────────────────────────────────────────────────────

# Путь к бинарнику frpc
MYXON_FRPC_BIN=/usr/local/bin/frpc

# Файл frpc-токена устройства — управляется автоматически, не редактировать
# MYXON_TOKEN_FILE=/etc/myxon/agent_token

# ─── Режим LAN-роутера ─────────────────────────────────────────────────────────

# USB Ethernet-адаптер как выделенный интерфейс LAN (Orange Pi как роутер/шлюз).
# Когда задан, агент сканирует ТОЛЬКО этот интерфейс — MYXON_SCAN_MODE игнорируется.
# Ставится автоматически через install.sh --lan-iface.
# MYXON_LAN_IFACE=eth1

# ─── 4G WAN-резерв + GSM SMS ───────────────────────────────────────────────────

# Порт USB GSM-модема. Служит двум целям:
#
#   1. WAN-РЕЗЕРВ    — NetworkManager / ModemManager используют этот модем как
#      резервный интернет (eth0 основной metric 100, wwan0 резерв metric 200).
#      Ставится автоматически при install.sh --backup-modem.
#
#   2. ЛОКАЛЬНЫЙ SMS — агент использует mmcli (CLI ModemManager) для отправки
#      SMS об авариях через этот модем. Откатывается к сырым AT-командам, если
#      mmcli недоступен. Интернет не нужен — SMS уходит прямо в GSM-сеть.
#
# Оставьте пустым, чтобы отключить локальный SMS (email-уведомления работают через SMTP сервера).
# MYXON_MODEM_PORT=/dev/ttyUSB0

# ─── Ручное переопределение ресурсов ───────────────────────────────────────────
# Пропустить авто-обнаружение LAN и использовать фиксированный список ресурсов.
# Формат JSON-массива. Полезно, когда IP контроллера статичен, а обнаружение медленно.
#
# MYXON_RESOURCES=[
#   {"id":"hmi","protocol":"tcp","host":"192.168.1.100","port":5843,"name":"HMI Panel"}
# ]
```

## Приоритет при загрузке

При каждом старте агент проверяет в таком порядке:

```
1. /etc/myxon/device.json существует?
   ДА → восстановить identity (серийник + токен), вызвать /register
   НЕТ ↓

2. MYXON_ACTIVATION_CODE задан?
   ДА → вызвать /activate, сохранить identity в device.json
   НЕТ ↓

3. MYXON_SERIAL задан?
   ДА → вызвать /register (legacy пред-регистрация)
   НЕТ → использовать дефолт "MYXON-DEV-001" (только для разработки)
```

## Systemd-сервис

Агент работает как systemd-сервис. Ключевые настройки:

```ini
[Service]
EnvironmentFile=/opt/myxon-agent/agent.env
ExecStart=/opt/myxon-agent/venv/bin/python myxon_agent.py
Restart=always
RestartSec=10
```

`Restart=always` означает, что агент автоматически восстанавливается после падений или сетевых сбоев. После обрыва он пере-регистрируется и заново поднимает туннель — без ручного вмешательства.

## Устранение неполадок

**Агент постоянно перезапускается:**
```bash
journalctl -u myxon-agent -n 100 --no-pager
```

**Туннель не подключается:**
```bash
# Проверить, что frpc установлен
frpc --version

# Проверить достижимость хоста frps (порт 7000)
nc -zv frps.myxon.yourcompany.com 7000
```

**Код активации отклонён:**
- Код истёк: сгенерируйте новый в Портале дилера
- Код уже использован: проверьте `/etc/myxon/device.json` — устройство, возможно, уже активировано
- Неверный формат кода: должен быть `XXXX-XXXX-XXXX-XXXX` (hex-цифры, верхний регистр)

**Обнаружение ничего не находит:**
```bash
# Проверить, какие интерфейсы доступны
ip -o -4 addr show

# Попробуйте режим 'all', если контроллеры в подсети WAN-интерфейса
# MYXON_SCAN_MODE=all
```

**SMS не доставляются:**
```bash
# Проверить, видит ли ModemManager модем
mmcli -L

# Проверить статус модема (замените 0 на индекс модема из -L)
mmcli -m 0

# Отправить тестовый SMS вручную
mmcli -m 0 --messaging-create-sms="number=+31612345678,text=Test from MYXON"
# Скопируйте возвращённый путь dbus (напр. /org/freedesktop/ModemManager1/SMS/0)
mmcli -s /org/freedesktop/ModemManager1/SMS/0 --send

# Проверить SIM и сигнал
mmcli -m 0 --signal-get
```

::: tip Модемы Huawei HiLink (E3372 и подобные)
Эти модемы выставляют **встроенный роутер** через USB Ethernet — они НЕ видны как
устройство `/dev/ttyUSB*` и НЕ управляются ModemManager. Работают автоматически как
второй Ethernet-интерфейс. `MYXON_MODEM_PORT` следует оставить пустым; SMS через
HiLink-модемы недоступен. Для SMS + данных на одном устройстве используйте
Quectel EC25 или модем Sierra Wireless.
:::
