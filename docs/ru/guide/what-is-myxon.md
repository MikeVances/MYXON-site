# Что такое MYXON?

MYXON — это **платформа удалённого доступа и мониторинга** для автоматизации сельского хозяйства. Она позволяет интеграторам, дилерам и их клиентам подключаться к фермерским контроллерам — системам климата, вентиляции, освещения — откуда угодно в мире, не трогая настройки firewall или роутера.

## Какую проблему решает

Аграрные контроллеры живут глубоко внутри фермерских сетей. Традиционно, чтобы достучаться до них удалённо, нужно:

- Пробрасывать порты на роутере фермы (нужны IT-знания, риск безопасности)
- Разворачивать VPN-инфраструктуру (дорого, сложно в поддержке)
- Выезжать на объект ради каждой правки конфигурации (время и деньги)

**MYXON убирает всё это.** Небольшой агент работает на Orange Pi (или на встроенном Linux-сервере контроллера) и создаёт исходящий шифрованный туннель в облако MYXON. Сеть фермы делает только исходящие подключения — входящие порты не нужны.

## Как это работает — одной картинкой

```
  Контроллер (Modbus/HMI)          Облако MYXON
  ┌────────────────────┐           ┌────────────────────────┐
  │  192.168.1.100     │           │  frps.myxon.io:7000    │
  │  порт 5843 (HMI)   │           │  Backend (FastAPI)      │
  └────────┬───────────┘           │  Frontend (React)       │
           │ LAN                   └───────────┬────────────┘
  ┌────────▼───────────┐                       │
  │  Orange Pi          │──── исходящий TCP ──▶│
  │  myxon_agent.py     │◀─── конфиг туннеля ──│
  │  frpc               │                       │
  └────────────────────┘                       │
                                               │ HTTPS
                                          ┌────▼──────────┐
                                          │  Браузер       │
                                          │  (дилер или   │
                                          │   клиент)     │
                                          └───────────────┘
```

## Ключевые роли

| Роль | Кто это | Что делает в MYXON |
|------|---------|--------------------|
| **Администратор платформы** | Команда MYXON | Управляет облачной инфраструктурой |
| **Дилер** | Интегратор / реселлер / OEM-производитель | Регистрирует устройства, приглашает клиентов, следит за парком |
| **Клиент** | Владелец / оператор фермы | Активирует устройства, следит за авариями, открывает HMI-сессии |
| **Инженер** | Техник фермы (сотрудник клиента) | Смотрит устройства, открывает подключения, видит аварии |
| **Наблюдатель** | Управляющий фермой | Только просмотр аварий |

## Два сценария развёртывания

MYXON поддерживает две бизнес-модели:

### Сценарий 1 — MYXON Box

Вы (дилер) продаёте готовый Orange Pi-роутер с предустановленным агентом. Вы регистрируете каждый серийный номер в портале дилера, печатаете QR-этикетку и клеите её на бокс. Клиент сканирует QR, чтобы забрать и активировать устройство.

→ См. [Дилер: Регистрация устройств](/ru/dealer/register-devices)

### Сценарий 2 — OEM SDK

Ваша компания уже выпускает контроллеры со встроенным Linux-сервером (как Orange Pi). Вы ставите агента MYXON однострочным установщиком и настраиваете его одноразовым кодом активации, сгенерированным в портале дилера.

→ См. [Интеграция OEM / SDK](/ru/oem/overview)
