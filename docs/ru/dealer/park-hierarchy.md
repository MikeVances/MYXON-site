# Многоуровневая иерархия парка

Организуйте оборудование так, как реально устроен ваш бизнес — от отдельного помещения до
холдинга из нескольких юрлиц. MYXON моделирует весь парк как гибкое дерево, поэтому и небольшое
хозяйство, и крупный агрохолдинг ощущаются одинаково естественно.

## Дерево локаций — любой глубины

Группируйте устройства в дерево любой глубины: **фабрика → площадка → помещение → устройство**.
Каждый уровень — просто метка: маленькому хозяйству хватит одного узла, холдинг строит несколько
уровней.

| Возможность | Что даёт |
|-------------|----------|
| **Вложенные локации** | Навигация по парку как по дереву — ветки разворачиваются и сворачиваются |
| **Роллап тревог** | Тревога на устройстве загорается на его узле **и всех родителях** — сразу видно, в какой ветке проблема, не разворачивая дерево |
| **Наследование настроек** | Правило уведомлений или доступ на узле действуют на всё его поддерево («директор фабрики получает тревоги всех залов») |

::: tip Роллап тревог
Здесь MYXON выделяется: тревога глубоко в дереве всплывает на каждый уровень выше. Большинство
инструментов удалённого доступа оставляют группировку плоской — проблему пришлось бы искать.
:::

## Обзор холдинга — по юрлицам

Для групп из нескольких юридических лиц MYXON даёт **единое окно** над всеми сразу:

- Один дашборд с итогами по всем юрлицам: устройств онлайн, активных тревог, площадок.
- Разбивка по юрлицам — видно, где нужно внимание.
- Данные каждого юрлица **изолированы** — холдинг видит свои юрлица; юрлица не видят друг друга.
  Сводка доступна только владельцу холдинга.

::: info Две независимые структуры
**Дерево локаций** организует оборудование внутри одного юрлица. **Холдинг** агрегирует по юрлицам.
Это разные оси — юрлицо может входить и в дилерский канал, и в холдинг одновременно.
:::
