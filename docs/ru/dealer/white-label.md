# White-label и брендированный вход

Портал под **вашим брендом**. Ваши клиенты видят ваш логотип, ваши цвета и ваше название —
а не MYXON. Идеально для интеграторов, реселлеров и OEM-производителей, которым нужен
цельный опыт для конечных клиентов.

> **Screenshot:** *Страница входа с логотипом партнёра, фирменным акцентным цветом и подписью — на выделенном субдомене*

## Что можно настроить

| Элемент | Описание |
|---------|----------|
| **Логотип** | Ваш логотип на странице входа и в шапке |
| **Фирменные цвета** | Акцент кнопок и ссылок, а также фон страницы входа |
| **Название и подпись** | Название компании и подпись под логотипом |
| **Favicon** | Иконка вкладки браузера |
| **Свой адрес** | Выделенный субдомен вида `вашбренд.<домен>` |

Любое незаполненное поле остаётся в стандартном оформлении MYXON — настраивайте столько,
сколько нужно.

## Как это выглядит для ваших клиентов

Клиент открывает ваш адрес (например `вашбренд.<домен>`) и сразу видит ваш логотип и цвета —
ещё до входа. Сам вход обычный: по паролю или — если настроено — через единую кнопку
корпоративного входа.

::: info Без установки приложений
Брендированный портал работает полностью в браузере. Вашим клиентам не нужно ничего устанавливать.
:::

## Корпоративный вход (SSO) для крупных клиентов

Если ваш клиент — крупный холдинг с корпоративной системой учётных записей (Active Directory,
Azure/Entra, Keycloak, ADFS), вход можно связать с ней:

- сотрудники входят своей **рабочей учётной записью** — без отдельного пароля MYXON;
- при включённом бесшовном входе на стороне холдинга доступ может быть **полностью
  автоматическим** — без запроса логина, сразу после входа в Windows.

::: tip Вход без единого клика
Когда корпоративная система поддерживает бесшовный вход на рабочих станциях, открытие
брендированного адреса сразу авторизует пользователя — без кликов и ввода.
:::

## Как заказать

Брендирование и корпоративный вход настраивает **платформенная команда MYXON** по запросу.
Подготовьте:

1. логотип и favicon (SVG или PNG);
2. фирменные цвета (hex-коды);
3. желаемое название и подпись;
4. предпочтительный субдомен;
5. для SSO — контакт ИТ/ИБ вашего клиента для настройки доверия с его системой учётных записей.
