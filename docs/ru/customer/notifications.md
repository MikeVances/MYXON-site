# Уведомления об авариях

MYXON автоматически уведомляет нужных людей, когда срабатывает авария — по **email** и/или **SMS через GSM-модем** — не требуя ручной проверки панели.

## Как это работает

```
Контроллер шлёт аварию
    │
    ▼
MYXON собирает и сохраняет аварию
    │
    ├─► Email подходящим контактам   (сразу через SMTP)
    └─► SMS через GSM-модем устройства (доставка в пределах одного heartbeat, ~15 с)
```

Уведомления маршрутизируются **правилами**. Каждое правило связывает контакт с областью (весь тенант, одна площадка или конкретное устройство) и фильтрует по важности и категории аварии.

## Контакты

**Контакт** — это человек, который может получать уведомления. Один контакт может фигурировать во многих правилах.

| Поле | Описание |
|------|----------|
| **Имя** | Отображаемое имя — «Иван Петров (Инженер)» |
| **Телефон** | Формат E.164: `+31612345678` — для SMS |
| **Email** | Для email-уведомлений |
| **Каналы** | Какие методы использовать: SMS, Email или оба |
| **Активен** | Неактивные контакты никогда не получают уведомлений |

::: tip Формат телефона E.164
Всегда указывайте код страны: `+31` (Нидерланды), `+49` (Германия), `+7` (Россия).
Без пробелов и дефисов — только `+` и цифры.
:::

### Добавить контакт

Зайдите в **Настройки → Уведомления → Контакты** и нажмите **Добавить контакт**.

> **Скриншот:** *Вкладка «Контакты» с двумя записями: «Иван Петров (Инженер)» с телефоном и обоими каналами, и «Мария Смит (Менеджер)» с телефоном, только канал SMS.*

## Правила

**Правило** определяет: *когда* уведомлять *этот контакт*?

| Поле | Варианты | Описание |
|------|----------|----------|
| **Контакт** | из вашего списка контактов | Кто получает уведомление |
| **Область** | Тенант / Площадка / Устройство | Какие устройства запускают правило |
| **Мин. важность** | Warning+ / Только Alarm | Alarm срабатывает всегда; Warning+ срабатывает и на предупреждениях |
| **Категории** | пусто = все, или выбрать конкретные | напр. только `temperature` и `power` |
| **Активно** | вкл/выкл | Отключить временно без удаления |

### Что значит «Область»

```
Правило на весь тенант → срабатывает для ВСЕХ устройств организации
Правило на площадку    → срабатывает для ВСЕХ устройств одной фермы/локации
Правило на устройство  → срабатывает для ОДНОГО конкретного устройства
```

Несколько правил могут совпасть с одной аварией — все подходящие контакты получат уведомление, дубли убираются.

### Пример: команда фермы из двух человек

| Правило | Контакт | Область | Важность | Категории |
|---------|---------|---------|----------|-----------|
| 1 | Иван (Инженер) | Площадка: Ферма Норд | Warning+ | все |
| 2 | Мария (Менеджер) | Весь тенант | Только Alarm | power, communication |
| 3 | Команда поддержки | Весь тенант | Только Alarm | все |

С такой настройкой:
- Иван получает **все аварии и предупреждения** с Фермы Норд по SMS и email
- Мария получает **только критичные** (сбои питания и связи) со всех ферм
- Поддержка получает **все критичные аварии** по всему тенанту

### Добавить правило

Зайдите в **Настройки → Уведомления → Правила** и нажмите **Добавить правило**.

> **Скриншот:** *Вкладка «Правила» с тремя правилами: бейджи области (Весь тенант, Площадка, Устройство), чипы важности (Warning+ янтарным, Только Alarm красным) и кнопки правки/удаления.*

## Доставка SMS — как используется GSM-модем

SMS доставляются через **локальный GSM-модем на Orange Pi**, а не через интернет-шлюз. Это значит:

- SMS работают, даже когда интернет-связь деградировала
- Не нужен сторонний SMS-аккаунт или подписка
- Задержка доставки ≈ интервал heartbeat (по умолчанию 15 секунд)

```
Backend кладёт SMS в ответ heartbeat
    │
    └─► Агент получает её на следующем heartbeat (~15с)
            │
            └─► Агент шлёт через mmcli → GSM-модем → SMS-сеть
```

::: info Нет модема — нет SMS
Если устройство установили без `--backup-modem`, GSM-модем не настроен.
В этом случае доставятся только email-уведомления.
Уточните у OEM или установщика, если нужны SMS.
:::

## Email-уведомления

Email отправляется прямо с сервера MYXON при срабатывании аварии. Настройку делает администратор платформы в файле сервера `.env`:

```ini
SMTP_HOST=smtp.yourcompany.com
SMTP_PORT=587
SMTP_USER=noreply@yourcompany.com
SMTP_PASSWORD=...
SMTP_FROM=noreply@yourcompany.com
```

Оставьте `SMTP_HOST` пустым, чтобы глобально отключить email (режим только-SMS).

## Справочник важности аварий

| Важность | Цвет | Типичная причина |
|----------|------|------------------|
| **Alarm** | Красный | Критический сбой — нужны немедленные действия |
| **Warning** | Янтарный | Ненормальное состояние — внимательно следить |

Важности Off / Suppressed / None уведомлений не запускают.

## Категории аварий

Категории классифицируют источник аварии. Используйте их в правилах для фильтрации:

| Категория | Примеры |
|-----------|---------|
| `temperature` | Слишком высокая/низкая темп., предупреждение о заморозке |
| `humidity` | Влажность вне диапазона |
| `co2` | Авария концентрации CO₂ |
| `ventilation` | Отказ вентилятора, перекрыт воздухопоток |
| `pressure` | Авария давления |
| `weather` | Датчики ветра, дождя, шторма |
| `communication` | Потеря связи, таймаут сети |
| `power` | Сбой напряжения, ИБП, батарея |
| `sensor` | Отказ датчика, неверное показание |
| `general` | Всё остальное |

::: tip Правило большого пальца
Инженеры, реагирующие на объекте → подписка на все категории на «Warning+».
Менеджеры, эскалирующие → подписка на `power` и `communication` на «Только Alarm».
:::
