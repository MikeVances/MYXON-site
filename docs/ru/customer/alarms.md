# Аварии

MYXON собирает аварийные события с подключённых контроллеров и показывает их в единой панели аварий. Панель видна на каждой панели устройства и фильтруется по важности.

## Состояния аварии

```
  ACTIVE ──► ACKNOWLEDGED ──► CLEARED
    │                            │
    └── авто-сброс, если контроллер ──┘
        шлёт сигнал сброса
```

| Состояние | Значение | Цвет |
|-----------|----------|------|
| **Активна** | Авария срабатывает прямо сейчас | Красный |
| **Подтверждена** | Пользователь увидел и подтвердил | Янтарный |
| **Сброшена** | Условие аварии устранено | Зелёный |

## Панель аварий

> **Скриншот:** *Панель аварий на панели устройства: таблица со столбцами Важность (цветной бейдж), Код, Сообщение, Сработала, Подтверждена и кнопка «Подтвердить». Три строки: одна CRITICAL (красная), одна WARNING (янтарная, подтверждённая), одна INFO (серая, сброшенная).*

### Столбцы

| Столбец | Описание |
|---------|----------|
| **Важность** | `critical` / `warning` / `info` |
| **Код** | Числовой код аварии от контроллера |
| **Сообщение** | Человекочитаемое описание (если дал вендор) |
| **Сработала** | Когда авария впервые сработала |
| **Подтверждена** | Метка времени + кто подтвердил (пусто, если ещё нет) |

## Подтвердить аварию

Нажмите **Подтвердить** рядом с активной или неподтверждённой аварией. Это помечает её как увиденную и ставит метку `acknowledged_at` с вашим ID.

::: warning Нужна политика доступа
Подтверждение аварий требует разрешения `allow_alarms_acknowledge` в вашей политике доступа. Если кнопки «Подтвердить» нет, ваша роль или политика площадки ограничивает это действие. Обратитесь к админу.
:::

## Фильтр важности

Панель аварий по умолчанию показывает все важности. Если у вашей политики задан **фильтр важности** (напр. «warning и выше»), аварии уровня info автоматически скрываются для вас.

| Настройка фильтра | Что вы видите |
|-------------------|---------------|
| Все | Critical + Warning + Info |
| Warning и выше | Critical + Warning |
| Только critical | Только Critical |

## История аварий

Панель аварий на панели устройства показывает **текущие и недавние** аварии. Для исторических данных по авариям и отчётности по устройствам полный след событий даёт журнал аудита.

## Уведомления

MYXON может автоматически уведомлять вашу команду, когда сработала авария — по **email** или **SMS через GSM-модем** — не требуя, чтобы кто-то смотрел на панель.

Уведомления настраиваются в **Настройки → Уведомления**:
- Добавьте **контакты** (инженеры, менеджеры) с их телефоном и/или email
- Добавьте **правила**, определяющие, кого уведомлять по каким устройствам и при какой важности

→ [Контакты и правила уведомлений](/ru/customer/notifications)
