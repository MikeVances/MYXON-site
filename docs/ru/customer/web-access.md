# Веб-HMI в браузере

Многие аграрные контроллеры поставляются со своим **веб-интерфейсом** — встроенным HMI, отдаваемым по HTTP в локальной сети. Раньше добраться до него можно было только стоя у машины или подняв VPN.

MYXON рендерит этот веб-интерфейс **прямо в вашем браузере**, через тот же исходящий туннель, который уже держит агент. Ни VPN-клиента, ни проброса портов, ни выставленного наружу устройства.

> **Скриншот:** *Карточка «Веб-доступ» на панели устройства с HTTP-ресурсом «Controller UI (HTTP · порт 80)» и кнопкой «Открыть веб →».*

## Как это выглядит для вас

1. Откройте **панель устройства** и найдите раздел **Веб-доступ**.
2. Ресурсы, которые агент обнаружил как **веб-серверы** (HTTP), показывают кнопку **Открыть веб →**.
3. Нажмите её — MYXON запросит короткоживущий одноразовый доступ и откроет веб-интерфейс контроллера в **новой вкладке браузера**.

Вот и всё. Вы работаете с собственным HMI вендора ровно так, будто вы в локальной сети фермы.

::: info Почему новая вкладка?
Многие HMI контроллеров шлют заголовки, запрещающие встраивание в другую страницу (`X-Frame-Options`). Открытие в отдельной вкладке даёт полный, немодифицированный интерфейс.
:::

## Что делает это безопасным

- **Никаких входящих портов.** Браузер никогда не подключается к ферме. Он общается с MYXON, который релеит трафик через существующий исходящий туннель агента.
- **Одноразовые доступы с ограничением по времени.** Каждая сессия авторизуется отдельно и истекает автоматически (30 минут по умолчанию).
- **Фиксированная цель.** Прокси может достучаться только до конкретного сервиса, который опубликовал агент — его нельзя перенаправить на произвольные хосты в сети.
- **Под политикой.** Веб-доступ подчиняется тем же [политикам доступа](/ru/customer/access-policies), что и любой другой ресурс. Если ваша роль или политика площадки блокирует HTTP, кнопка **Открыть веб →** неактивна.

## Текущие ограничения (v1)

Веб-HMI работает лучше всего со **стандартными веб-интерфейсами на относительных путях**. Два случая требуют доработки и есть в дорожной карте:

- **Живые/стриминговые виджеты по WebSocket** пока не проксируются — страница загружается, но real-time push-элементы могут не обновляться.
- **SPA, запрашивающие ресурсы по абсолютным корневым путям** (`/app.js`), могут потребовать поддоменного проксирования на устройство — запланировано на будущий релиз.

Для этих случаев в качестве запасного варианта остаётся VNC-доступ.

## Что когда использовать

| Вы хотите… | Используйте |
|------------|-------------|
| Достучаться до **собственного веб-UI** контроллера | **Открыть веб →** (эта фича) |
| Увидеть **физический экран HMI** контроллера без веб-UI | [Прямой HMI / VNC](/ru/customer/device-dashboard) |
| Универсальный удалённый рабочий стол к edge-устройству | VNC |
