# Multi-Level Park Hierarchy

Organize equipment the way your business is actually structured — from a single room up to a
multi-entity holding. MYXON models the whole park as a flexible tree, so a small farm and a
large agro-holding both feel native.

> **Screenshot:** *Location tree — Factory ▸ House-1 / House-2 — with a red dot rolled up to the Factory node showing an active alarm below*

## Location tree — any depth

Group devices into a tree of any depth: **facility → site → room → device**. Each level is
just a label — a small operation uses one node, a holding builds several levels.

| Capability | What it gives |
|------------|---------------|
| **Nested locations** | Navigate the park as a tree — expand and collapse branches |
| **Alarm roll-up** | An alarm on a device lights up its node **and all parents** — see at a glance which branch is on fire, without expanding the tree |
| **Inherited settings** | A notification rule or an access grant placed on a node applies to its whole subtree ("factory manager gets alarms from all houses") |

::: tip Alarm roll-up
This is where MYXON stands out: an alarm deep in the tree surfaces at every level above it.
Most remote-access tools keep grouping flat — you'd have to hunt for the problem.
:::

## Holding overview — across legal entities

For groups of several legal entities, MYXON gives a **single window** over all of them:

- One dashboard with totals across every entity: devices online, active alarms, sites.
- A per-entity breakdown so you can see where attention is needed.
- Each entity's data stays **isolated** — a holding sees its own entities; entities don't see
  each other. The aggregate is visible only to the holding owner.

::: info Two independent structures
The **location tree** organizes equipment inside one legal entity. The **holding** aggregates
across entities. They're separate axes — an entity can belong to a dealer channel *and* a
holding at the same time.
:::
